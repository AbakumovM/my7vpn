# Frontend Design Spec — MY7VPN Personal Cabinet

**Date:** 2026-05-08
**Status:** Approved

---

## Overview

React + Vite + TypeScript SPA в папке `frontend/`. Личный кабинет для веб-пользователей VPN-сервиса. Работает поверх существующего FastAPI бэкенда. Responsive: десктоп и мобильный.

**Деплой:**
- Dev: Vite dev server (`:5173`) проксирует `/api/*` → FastAPI (`:8000`)
- Prod: nginx раздаёт `frontend/dist/` как статику, проксирует `/api/*` → FastAPI контейнер

## Design System

### Палитра

| Переменная | Значение | Применение |
|-----------|----------|-----------|
| `--black` | `#000000` | Фон страницы |
| `--surface` | `#0D0D0D` | Карточки, блоки |
| `--surface-2` | `#1C1C1C` | Hover-состояния, вложенные блоки |
| `--border` | `#282828` | Разделители, рамки |
| `--white` | `#FFFFFF` | Основной текст, кнопки (активные) |
| `--muted` | `#444444` | Лейблы, подписи, неактивные элементы |
| `--dim` | `#222222` | Разделители сетки |

Интерактивные состояния — только через яркость белого:
- Активная вкладка / выбранный элемент: белый текст + белая черта
- Неактивные элементы: `--muted`
- Hover: переход к белому

Статусы через оттенки (без цвета):
- Активна: `rgba(255,255,255,0.9)` с точкой `●`
- Истекает скоро (≤7 дней): курсив или пульсация
- Нет подписки: `--muted` текст

### Типографика

| Использование | Шрифт | Вес | Замечания |
|--------------|-------|-----|-----------|
| Заголовки страниц (`h1`) | Cormorant Garamond | 300 | Крупный, тонкий |
| Числа / данные (дни, суммы) | Cormorant Garamond | 600 | Выразительные цифры |
| Лейблы, кнопки, навигация | JetBrains Mono | 400/700 | `text-transform: uppercase`, широкий letter-spacing |
| Мелкий служебный текст | JetBrains Mono | 300 | color: `--muted` |

### Фоновая текстура

Тонкая архитектурная сетка через `body::before`:
```css
background-image:
  linear-gradient(to right, rgba(255,255,255,0.018) 1px, transparent 1px),
  linear-gradient(to bottom, rgba(255,255,255,0.018) 1px, transparent 1px);
background-size: 80px 80px;
```

### Карточки

Левая вертикальная черта `2px solid #fff` — фирменный элемент. Все информационные блоки имеют эту черту слева.

### Кнопки

- **Primary**: белый фон, чёрный текст, clip-path со срезанным углом (8px)
- **Ghost**: прозрачный, белая рамка `1px solid --border`, hover → `1px solid --white`
- **Disabled**: opacity 0.35

## Экраны

### 1. Логин (`/login`)

Центрированная карточка на весь экран. Два шага:

**Шаг 1 — Email:**
- Лого (SVG аватара) + название
- Поле email
- Кнопка «Получить код»

**Шаг 2 — OTP:**
- Текст «Отправили на {email}»
- Поле ввода кода (крупный моноширинный, текст по центру, letter-spacing: 0.4em)
- Кнопка «Войти»
- Ссылка «Изменить email»

Подсказка внизу: «Или войдите через бот командой /web»

### 2. Дашборд (`/dashboard`) — Вариант B одобрен

**Главный блок — счётчик дней:**
```
┌─────────────────────────────┐
│   ПОДПИСКА АКТИВНА          │
│                             │
│          42                 │  ← Cormorant 72px
│       дня осталось          │
│                             │
│  ████████████░░░░  65%      │  ← прогресс-бар
│  до 15 июня 2026            │
└─────────────────────────────┘
```

**Строка метрик (3 колонки):**
```
┌──────────┬──────────┬──────────┐
│ Устройств│  Баланс  │Рефералов │
│    2     │  150 ₽   │    3     │
└──────────┴──────────┴──────────┘
```

**Блок подключения** (если есть `subscription_url`):
- Input (readonly) + кнопка «Копировать»
- QR-код: генерируется клиентски из `subscription_url` через `qrcode.react` — никаких доп. запросов к бэкенду
- Кнопка «Показать QR» / «Скрыть QR» (toggle)
- Инструкция под блоком:
  ```
  1. Скопируйте ссылку или отсканируйте QR
  2. Откройте Hiddify / NekoBox / v2rayNG
  3. «Добавить подписку» → вставьте ссылку
  ```

**Состояния:**
- Нет подписки → вместо счётчика CTA-блок «Подключить VPN → Выбрать план»
- Истекает ≤7 дней → предупреждение с кнопкой «Продлить»

### 3. Подписка / Оплата (`/payment`) — Вариант A одобрен

**Шаг 1 — Устройства** (3 кнопки-переключателя):
```
[ 1 устройство ] [ 2 устройства ] [ 3 устройства ]
  (выбран = белый фон)
```

**Шаг 2 — Период** (сетка 2×2):
```
┌──────────────┬──────────────┐
│  1 месяц     │  3 месяца    │
│  150 ₽       │  400 ₽ −10% │
├──────────────┼──────────────┤
│  6 месяцев   │  1 год       │
│  750 ₽ −15% │ 1200 ₽ −20% │
└──────────────┴──────────────┘
```
Выбранный тариф — белый фон + белая рамка. Остальные — тёмный фон.

**Блок итога:**
- «Итого: 150 ₽» + «Баланс покроет: 150 ₽» (если есть) + «К оплате: 0 ₽»
- Кнопка «Оплатить →» / «Активировать» (если 0 ₽)

**Состояния после оплаты:**
- `final_amount > 0`: редирект на YooKassa, потом polling страница (лоадер + «Ожидание оплаты»)
- `final_amount == 0`: мгновенное подтверждение, экран «Активировано ⚡»

### 4. История платежей (`/history`)

Таблица (на мобильном — карточки):

| Дата | Сумма | Период | Устройства | Метод |
|------|-------|--------|-----------|-------|

Итоговая строка внизу: «Всего платежей: N | Потрачено: X ₽»

Пустое состояние: центрированный текст «Платежей пока нет»

### 5. Рефералы (`/referral`)

**Блок реферальной ссылки:**
- Ссылка (input readonly) + кнопка «Копировать»
- Счётчик: «Приглашено: N человек»

**Объяснение механики:**
- «Пригласи друга → он получает бесплатный месяц → ты получаешь 50 ₽ на баланс»

## Роутинг

```
/login        → LoginPage (публичный)
/dashboard    → DashboardPage (требует JWT)
/payment      → PaymentPage (требует JWT)
/history      → HistoryPage (требует JWT)
/referral     → ReferralPage (требует JWT)
/             → redirect → /dashboard
```

401 от API → автоматический редирект на `/login` (через axios interceptor).

## Деплой (production)

`nginx.conf` в корне проекта:
```nginx
location /api/ {
  proxy_pass http://fastapi:8000;
}
location / {
  root /app/frontend/dist;
  try_files $uri /index.html;
}
```

`docker-compose.prod.yml` добавляется сервис `frontend` (multi-stage build: node → nginx).
