# Bot Improvements Plan

Анализ проведён 2026-04-10. Задачи разбиты по приоритету.

---

## P0 — Критичные баги

### [ ] 1. Транзакция на создание устройства + списание баланса
**Проблема:** Создание устройства и списание баланса не атомарны. При сбое сети устройство создаётся, а баланс не списывается (или наоборот).
**Файл:** `src/apps/device/controllers/bot/router.py` ~строка 379
**Решение:** Объединить создание устройства и списание баланса в одну транзакцию через единый UoW. Оба действия должны либо пройти вместе, либо откатиться вместе.

---

### [ ] 2. Идемпотентность кнопки "Оплачено"
**Проблема:** Пользователь нажимает кнопку "Оплачено" дважды → создаётся два устройства.
**Файл:** `src/apps/device/controllers/bot/router.py` ~строка 360
**Решение:** После первого нажатия редактировать сообщение (убирать кнопки) до завершения обработки. Либо проверять существование устройства с таким именем перед созданием.

---

### [ ] 3. Небезопасный парсинг callback_data
**Проблема:** `call.data.split(":")[2]` — падает с IndexError при неожиданных данных.
**Файлы:**
- `src/common/bot/router.py` ~строка 48
- `src/apps/device/controllers/bot/router.py` ~строки 122, 148
**Решение:** Заменить все строковые `split(":")` на типизированные `CallbackData`-классы Aiogram 3. Добавить try/except там, где невозможно сразу переделать.

---

## P1 — Продакшн-готовность

### [ ] 4. Redis FSM Storage вместо MemoryStorage
**Проблема:** При рестарте бота все FSM-состояния теряются. Пользователи в середине флоу (например, ввода email) получают сломанный UX.
**Файл:** `main_bot.py`
**Решение:** Подключить `RedisStorage` из `aiogram-redis-storage`. Добавить Redis в `docker-compose.yml` и `config.py`.

---

### [ ] 5. ThrottlingMiddleware (анти-флуд)
**Проблема:** Нет защиты от спама — пользователь может заспамить хендлеры.
**Решение:** Добавить стандартный `ThrottlingMiddleware` (через `aiogram-throttling` или вручную через `cachetools`). Лимит: 1 запрос / секунда на пользователя.

---

### [ ] 6. Глобальный error handler
**Проблема:** Необработанные исключения роняют обработчик без уведомления пользователя.
**Решение:** Добавить `@router.errors()` хендлер, который:
- Логирует исключение через structlog
- Отправляет пользователю понятное сообщение ("Что-то пошло не так, попробуй ещё раз")
- Уведомляет админа о критических ошибках

---

### [ ] 7. Webhook вместо polling
**Проблема:** Polling неэффективен — постоянные запросы к Telegram API. FastAPI уже есть в проекте.
**Решение:** Настроить webhook endpoint в `main_web.py` через `aiogram`'s `SimpleRequestHandler`. Убрать `dp.start_polling()` из `main_bot.py`.

---

## P2 — Качество кода

### [ ] 8. Унификация всех callback на CallbackData
**Проблема:** Два подхода смешаны — `VpnCallback` (типизированный) и строки `f"conf:{id}"`, `f"appr_del_device:{id}"`. Создаёт риск ошибок и усложняет поддержку.
**Файлы:** `src/common/bot/cbdata.py`, все роутеры
**Решение:** Создать отдельные `CallbackData`-классы для каждого действия:
- `DeviceDetailCallback(device_id: int)`
- `DeviceDeleteCallback(device_id: int)`
- `DeviceErrorCallback(device_id: int)`
- `SettingsCallback(platform: str)`

---

### [ ] 9. Вынос констант в config
**Проблема:** Magic numbers по всему коду — `50` (бонус рефералу), `7` (дней пробного периода), `150/400/700/1200` (цены тарифов).
**Решение:** Добавить в `config.py` секцию `BillingConfig`:
```python
referral_bonus: int = 50
free_trial_days: int = 7
prices: dict[int, int] = {1: 150, 3: 400, 6: 700, 12: 1200}
```

---

### [ ] 10. Удаление неиспользуемого RegisterVpn FSM
**Проблема:** `states.py` содержит `RegisterVpn` с двумя состояниями — нигде не используется.
**Файл:** `src/common/bot/states.py`
**Решение:** Удалить `RegisterVpn` полностью. Оставить только `EmailInput`.

---

### [ ] 11. ResetStateMiddleware — предупреждать пользователя
**Проблема:** Любая команда (`/help`, `/start`) молча очищает FSM-состояние. Пользователь в процессе ввода email теряет контекст без объяснения.
**Файл:** `main_bot.py` класс `ResetStateMiddleware`
**Решение:** Перед очисткой состояния — отправлять сообщение "Действие отменено" если было активное состояние.

---

## Низкий приоритет / На будущее

- **Retry для scheduler** — если уведомление об истечении подписки не доставлено, сейчас оно теряется
- **Inline mode** — для шаринга реферальных ссылок
- **BotCommandScopeAllPrivateChats + admin scope** — разные команды для юзеров и админа
- **i18n** — тексты сейчас размазаны по `lexicon.py` и прямо в коде
