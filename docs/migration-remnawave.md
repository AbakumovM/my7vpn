# Миграция на Remnawave — план работ

## Контекст и решения

**Текущая модель:** подписка привязана к устройству (1 девайс = 1 ключ в 3x-ui)
**Новая модель:** подписка на пользователя (1 аккаунт = N устройств + все серверы)

**Стратегия перехода:** dual-track в одном боте
- Старые юзеры живут на 3x-ui до истечения подписки — не трогаем
- При продлении или добавлении нового девайса → бот переводит в новый flow
- Когда все старые подписки истекут → убираем старый код

**Почему не два отдельных бота:**
- Плохой UX (редирект в другой бот)
- Двойная поддержка в переходный период
- Потеря истории/баланса пользователя

## Этап 3 — Новый flow в боте

**Статус: ✅ выполнен (2026-04-21)**

### Новый flow покупки

```
Выбор кол-ва устройств (1 / 3 / 5)
    ↓
Выбор срока (1 / 3 / 6 месяцев)
    ↓
Показ цены из матрицы тарифов
    ↓
Оплата (через админа, как сейчас)
    ↓
Создание Remnawave User → subscription URL
    ↓
Отправка ссылки + инструкция как добавить в приложение
```

- [x] Шаг 1.5 в боте: выбор количества устройств (1/2/3)
- [x] `TARIFF_MATRIX` — матрица цен (устройства × срок)
- [x] `subscription_url` отправляется пользователю после подтверждения оплаты
- [x] Старые юзеры мигрируют автоматически при первом продлении (`remnawave_uuid is None` → create_user)
